# Notion 쓰기 (회의록·팀 공용 문서 전용)

> **역할 분담**: 작업 컨텍스트의 정본은 **Obsidian vault 하나**다(obsidian-vault-manager
> 스킬). claude.md·context.md·지식 문서·history는 전부 vault에 있고, **Notion에는 그런
> 레이어가 없다.** 이 스킬은 Notion에 회의록과 팀 공용 문서를 쓰는 절차만 다룬다.
> 대상을 명시하지 않은 일반 저장 요청이 이 스킬로 들어왔다면 obsidian-vault-manager로 넘긴다.

## 이 스킬이 쓰는 것

1. **회의록** — 사용자가 지정한 회의록 페이지/DB에 작성.
2. **팀 공용 문서** — 팀이 함께 보는 Notion 문서의 작성·갱신.
3. **"Notion에 ○○ 써줘"** — 사용자가 대상 페이지를 명시한 쓰기 요청.

그 외(개인 규칙, 프로젝트 상태, 지식 정리, 대화 이력)는 전부 Obsidian 담당이다.
그런 내용을 Notion에 써 달라는 요청이 오면, 정본은 vault임을 알리고 vault에 기록한
뒤 필요 시 공유용 사본만 Notion에 만든다.

## 대상 페이지 결정

- 대상 페이지 URL은 **사용자가 준 것만** 쓴다. 메모리·추측 금지.
- URL이 없으면 어디에 쓸지 한 번 확인한다(회의록 DB, 부모 페이지 등).
- 이전 대화에서 확인된 페이지를 이어서 쓰는 것은 허용.

## 접근 범위 규율 (워크스페이스 검색 방지)

- 접근 가능한 URL = 사용자가 지정한 페이지 + 그 하위.
- `notion-search`·`notion-query` 계열 워크스페이스 탐색은 **자동으로 하지 않는다.**
  필요하면 "다른 곳에서 데이터를 가져올까요?"라고 인가를 요청하고, 명시적 승인이
  있을 때만 실행한다. (Cowork에서는 플러그인 PreToolUse 훅이 이를 강제해 승인
  프롬프트가 뜬다. chat에서는 이 지시만 적용된다.)
- 대상 페이지가 비어 있으면 그 사실을 알린다. 쓰기 작업이라면 첫 내용을 채우는
  것은 정상 진행이다(외부 탐색이 아니므로 인가 불필요).

## 회의록 형식 (기본값, 팀 양식이 있으면 그것을 따른다)

```
# YYYY-MM-DD <회의명>
- 참석: / 목적:
## 논의 내용
## 결정 사항
## 액션 아이템 (담당·기한)
```

- 회의의 **결정 사항이 프로젝트 컨텍스트에 영향**을 주면, 회의록 작성 후
  obsidian-vault-manager로 vault(해당 프로젝트 context/history)에도 반영할지 제안한다
  (중복 기록이 아니라 요약 반영).

## 공통 쓰기 규칙

1. **쓰기 전 diff 요약 + 승인.** 어느 페이지의 어디에 무엇을 추가·수정할지 짧게
   보여주고, 승인 전에는 Notion에 쓰지 않는다.
2. **중복 방지.** 쓰기 전 대상 페이지를 fetch해 현재 상태를 확인하고, 겹치면 새로
   추가하지 말고 기존 항목을 갱신한다.
3. **정밀 수정.** 해당 부분만 바꾸고 전체를 덮어쓰지 않는다.
4. 원문(대화 raw)을 그대로 붙여넣지 않고 요약·정리한다.

## 하지 말 것

- 승인 없이 Notion에 쓰기.
- 규칙·상태·지식·이력을 Notion에 저장(정본은 Obsidian vault).
- 사용자 인가 없이 워크스페이스 검색/쿼리.
- 사용자가 지정하지 않은 페이지에 쓰기.
